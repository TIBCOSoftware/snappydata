org.apache.spark.sql.execution.bootstrap

ResamplePlaceholder

Related Doc: package bootstrap

case class ResamplePlaceholder(child: SparkPlan) extends SparkPlan with UnaryExecNode with Product with Serializable

Remark: 1. Online planning should be applied before inserting exchange (and thus we exclude exchange here), because: (1) ImplementJoin will change join types, which will affect exchange; (2) Pushing resampling above exchange is always subsumed by pushing it above join. Note that pushing above exchange/join will change the order of tuples, and thus cannot be applied to multi-instance relations. 2. Turn off LeftSemiJoinHash for now, using Join instead.
